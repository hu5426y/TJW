# Open：项目开发记录

## 一、需求分析
### 1. 项目目标
建设社区智慧养老服务平台，支持多角色协同，实现服务供需对接、健康监护与安全保障。

### 2. 角色与职责
- **老年用户端**：预约服务、健康数据上传、紧急呼叫、投诉反馈。
- **家属端**：代预约、查看健康与服务状态、协助监督。
- **服务商端**：发布服务、处理预约、跟踪执行。
- **社区管理端**：监管全流程、处理投诉、应急联动。

### 3. 关键功能拆解
- 认证授权：注册、登录、角色权限控制。
- 服务管理：服务发布、服务列表、预约单创建。
- 健康监测：生命体征记录、阈值预警。
- 应急处置：一键呼叫、消息推送。
- 服务质控：投诉受理、状态管理。

---

## 二、技术路线与架构落地
### 1. 架构
- 前后端分离：Vue 前端 + SpringBoot REST API。
- 数据层：MySQL 持久化，Redis 缓存热点状态。
- 实时通信：WebSocket/STOMP推送健康预警与应急消息。

### 2. 主要依赖决策
- Spring Security + JWT：无状态认证授权。
- MyBatis-Plus：减少CRUD样板代码。
- Element Plus：快速搭建统一、适老化UI。

### 3. 数据库设计重点
- 用户表、服务表、预约表、健康记录表、投诉表、应急呼叫表。
- 关键字段规范：角色、状态、时间戳、流程轨迹。
